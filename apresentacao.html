Introdução

O QUE É O SISTEMA?

É uma pipeline de engenharia de dados responsável por transformar portfólios em PDF (não estruturados) em dados estruturados no formato CSV, prontos para ingestão em ambiente analítico (ClickHouse / GAIO).

Motivação

POR QUE O SISTEMA FOI CRIADO?

Os currículos exportados são documentos textuais livres. Sem estruturação, não é possível realizar consultas SQL, gerar dashboards ou cruzar dados com bases internas. O sistema automatiza esse processo reduzindo erros manuais e padronizando o modelo de dados.

Visão Geral

COMO O SISTEMA FUNCIONA?

O sistema opera em três etapas desacopladas:

01

Extração

PDF → TXT — Leitura e extração do conteúdo textual dos currículos em PDF.

02

Organização

Validação em lote — Processamento em massa com separação de erros e sucessos.

03

Estruturação

TXT → CSV — Enriquecimento via OpenAI com classificação de importância.

PRIMEIRA ETAPA

PDF → TXT (src/pdf_to_txt.py)
  • Recebe --input e --output via argparse
  • Cria logger dedicado em data/logs/{pdf}_{timestamp}.log
  • Leitura do PDF com PyPDF2.PdfReader e concatena page.extract_text() de cada página
  • Se não houver texto retorna status=1, caso contrário grava {pdf}.txt em data/txt/
$ python src/pdf_to_txt.py --input data/raw_pdfs/candidato.pdf --output data/txt
2

SEGUNDA ETAPA

Lote e Organização (src/batch_processor.py)
  • Varre data/raw_pdfs/*.pdf
  • Para cada PDF, executa pdf_to_txt.py via subprocess.run()
  • Se returncode==0, move para data/comp_pdfs, caso contrário move para data/quarantine
  • Garantia: cria as pastas comp_pdfs, quarantine e txt caso elas não existam
$ python src/batch_processor.py
3

TERCEIRA ETAPA

TXT → CSV + enriquecimento (main.py)

Objetivo: consolidar 1 linha por TXT e preencher campos com máxima confiabilidade, chamando OpenAI apenas quando necessário.

  • Detecta BASE_DIR (data ou Data), cria data/output/ e data/cache/
  • Lista TXTs em data/txt/*.txt; se não existir, aborta com diagnóstico
  • Carrega cache.jsonl (append-only) para reaproveitar resultados por SHA1 do texto
3

TERCEIRA ETAPA

Processamento por TXT (Pipeline)
  • Cache: se SHA1 existe no cache, usa Participant do cache (cache hit)
  • Parse KV: tenta extrair linhas 'campo: valor' (normalize_key + parse_kv)
  • OpenAI (condicional): se faltar nome+linkedin+empresa+cargo+headline, chama responses.parse com schema Participant
  • Registro: gera um arquivo .csv no caminho data/output/{arquivo}.csv
  • Arquivos finais: colunas do Participant — auditoria do LinkedIn (linkedin_txt, linkedin_norm, status, email_final, telefone_final)
$ python main.py

Pipeline

PIPELINE DENTRO DO GAIO

Este é um pipeline usado para filtrar a quantidade de pessoas por cada estado baseado no DDD do seu número de telefone.

🗄️
curriculos_raw
tabela origem
→
🔍
toString(Tel)
substring(1,2)
→
🏷️
CASE DDD
41=PR / 49=SC
→
📊
curriculos_ddd
+ coluna Estado
CREATE TABLE curriculos_ddd
ENGINE = MergeTree() ORDER BY tuple()
AS SELECT *,
  CASE substring(toString(Telefone), 1, 2)
    WHEN '41' THEN 'Paraná'
    WHEN '49' THEN 'Santa Catarina'
    ELSE 'Outro'
  END AS Estado
FROM curriculos_raw
